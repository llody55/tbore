# TBORE Pro 使用指南

## 概述

本指南详细说明了TBORE Pro系统的使用方法，包括设备管理、账户管理、隧道管理和WebSSH连接等功能的操作步骤。

## 登录系统

1. **访问系统**

打开浏览器，访问 `http://localhost:7835`

2. **登录界面**

在登录界面输入默认管理员账户信息：
- 用户名：admin
- 密码：admin123

3. **进入系统**

点击"登录"按钮，进入系统主界面。

## 设备管理

### 1. 查看设备列表

登录系统后，默认进入设备管理页面。在设备列表中，可以查看所有已注册的设备信息，包括设备ID、名称、Agent ID、IP地址、操作系统、状态、最后心跳时间和创建时间。

### 2. 注册设备

设备注册有两种方式：

#### 2.1 通过API注册

使用POST请求调用 `/api/v1/devices/register` 端点，提交设备信息：

```bash
curl -X POST http://localhost:7835/api/v1/devices/register \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "test-agent-123",
    "name": "Device-test-age",
    "ip_address": "192.168.1.100",
    "os": "Linux"
  }'
```

#### 2.2 通过Agent自动注册

在设备上安装并运行TBORE Agent，Agent会自动向服务器注册设备信息。

### 3. 编辑设备

1. 在设备列表中，找到需要编辑的设备。
2. 点击设备操作栏中的"更多"下拉菜单。
3. 选择"编辑信息"选项。
4. 在弹出的编辑窗口中，修改设备信息。
5. 点击"保存"按钮，完成设备编辑。

### 4. 删除设备

1. 在设备列表中，找到需要删除的设备。
2. 点击设备操作栏中的"更多"下拉菜单。
3. 选择"删除设备"选项。
4. 在弹出的确认窗口中，点击"确定"按钮，完成设备删除。

### 5. 绑定账户到设备

1. 在设备列表中，找到需要绑定账户的设备。
2. 点击设备操作栏中的"更多"下拉菜单。
3. 选择"绑定账户"选项。
4. 在弹出的绑定窗口中，选择要绑定的账户。
5. 点击"绑定"按钮，完成账户绑定。

### 6. 解除账户绑定

1. 在设备列表中，找到需要解除账户绑定的设备。
2. 点击设备操作栏中的"更多"下拉菜单。
3. 选择"解除绑定"选项。
4. 在弹出的解除绑定窗口中，选择要解除绑定的账户。
5. 点击"解除绑定"按钮，完成账户解除绑定。

### 7. 查看设备状态

在设备列表中，可以通过"状态"列查看设备的当前状态：
- 绿色：在线
- 红色：离线

### 8. 批量操作

1. 在设备列表中，勾选需要操作的多个设备。
2. 点击页面顶部的"批量删除"按钮，可以批量删除选中的设备。

## 账户管理

### 1. 进入账户管理页面

在左侧导航栏中，点击"账户管理"选项，进入账户管理页面。

### 2. 创建账户

1. 在账户管理页面，点击"添加账户"按钮。
2. 在弹出的添加账户窗口中，填写账户信息：
   - 账户名称
   - 用户名
   - 密码
   - 认证类型（默认密码）
   - 描述
   - 是否激活
   - 是否特权账户
3. 点击"保存"按钮，完成账户创建。

### 3. 编辑账户

1. 在账户列表中，找到需要编辑的账户。
2. 点击账户操作栏中的"编辑"按钮。
3. 在弹出的编辑窗口中，修改账户信息。
4. 点击"保存"按钮，完成账户编辑。

### 4. 禁用/启用账户

1. 在账户列表中，找到需要禁用/启用的账户。
2. 点击账户操作栏中的"禁用"或"启用"按钮。
3. 系统会自动切换账户状态。

### 5. 删除账户

1. 在账户列表中，找到需要删除的账户。
2. 点击账户操作栏中的"删除"按钮。
3. 在弹出的确认窗口中，点击"确定"按钮，完成账户删除。

### 6. 查看账户列表

在账户管理页面，可以查看所有已创建的账户信息，包括账户ID、名称、用户名、认证类型、描述、状态和创建时间。

## 隧道管理

### 1. 进入隧道管理页面

在左侧导航栏中，点击"隧道管理"选项，进入隧道管理页面。

### 2. 创建隧道

1. 在隧道管理页面，点击"添加隧道"按钮。
2. 在弹出的添加隧道窗口中，填写隧道信息：
   - 设备ID（选择设备）
   - 本地端口
   - 远程端口
   - 协议（默认TCP）
   - 描述
3. 点击"保存"按钮，完成隧道创建。

### 3. 查看隧道列表

在隧道管理页面，可以查看所有已创建的隧道信息，包括隧道ID、设备名称、本地端口、远程端口、协议、状态、描述和创建时间。

### 4. 启动/停止隧道

1. 在隧道列表中，找到需要启动/停止的隧道。
2. 点击隧道操作栏中的"启动"或"停止"按钮。
3. 系统会自动切换隧道状态。

### 5. 删除隧道

1. 在隧道列表中，找到需要删除的隧道。
2. 点击隧道操作栏中的"删除"按钮。
3. 在弹出的确认窗口中，点击"确定"按钮，完成隧道删除。

### 6. 查看设备的隧道

在设备管理页面，选择设备，点击"更多"下拉菜单，选择"查看隧道"选项，可以查看该设备的所有隧道。

## WebSSH连接

### 1. 打开Web终端

1. 在设备管理页面，找到需要连接的设备。
2. 点击设备操作栏中的"Web终端"按钮。
3. 系统会弹出Web终端登录窗口。

### 2. 选择登录方式

在Web终端登录窗口中，可以选择两种登录方式：

#### 2.1 使用绑定账户

1. 在"选择账号"下拉菜单中，选择已绑定到该设备的账户。
2. 系统会自动填充用户名和密码字段。

#### 2.2 手动输入账户信息

1. 在"选择账号"下拉菜单中，选择"手动输入"选项。
2. 在"用户名"和"密码"字段中，手动输入账户信息。

### 3. 配置连接参数

在Web终端登录窗口中，可以配置以下连接参数：
- 目标端口：SSH端口（默认22）
- 终端尺寸：终端的列数和行数（默认80×24）

### 4. 连接终端

点击"连接"按钮，系统会打开Web终端窗口，连接到设备。

### 5. 使用Web终端

在Web终端窗口中，可以执行以下操作：

- 输入命令：直接在终端中输入命令，按回车键执行。
- 复制粘贴：使用鼠标右键菜单或快捷键进行复制粘贴操作。
- 调整终端大小：拖动终端窗口边缘调整大小，终端会自动适应。

### 6. 关闭终端

点击终端窗口右上角的"关闭"按钮，关闭Web终端连接。

## 数据统计

### 1. 设备状态统计

在设备管理页面顶部，可以查看设备状态统计信息：
- 总设备数：所有已注册设备的数量
- 当前在线：当前在线设备的数量
- 异常/离线：当前离线设备的数量

### 2. 设备状态监控

在设备列表中，可以通过"状态"列和"最后心跳"列监控设备的在线状态：
- 在线设备：状态显示为绿色，最后心跳时间为最近时间
- 离线设备：状态显示为红色，最后心跳时间为较早时间

## 常见问题

### 1. 设备无法注册

**问题**：设备注册失败，返回错误信息。

**解决方案**：
- 检查网络连接是否正常
- 检查设备的Agent ID是否唯一
- 检查API请求参数是否正确

### 2. WebSSH连接失败

**问题**：WebSSH连接失败，终端显示连接错误。

**解决方案**：
- 检查设备是否在线
- 检查账户信息是否正确
- 检查SSH端口是否开放
- 检查设备的防火墙设置

### 3. 隧道无法启动

**问题**：隧道启动失败，状态显示为错误。

**解决方案**：
- 检查设备是否在线
- 检查本地端口是否被占用
- 检查远程端口是否开放
- 检查设备的防火墙设置

### 4. 账户绑定失败

**问题**：账户绑定到设备失败，返回错误信息。

**解决方案**：
- 检查账户是否存在
- 检查设备是否存在
- 检查API请求参数是否正确

### 5. 系统响应缓慢

**问题**：系统响应缓慢，操作卡顿。

**解决方案**：
- 检查服务器资源使用情况
- 检查网络连接是否正常
- 清理浏览器缓存
- 重启服务器

## 故障排查

### 1. 查看系统日志

系统日志位于服务器的日志目录中，可以通过查看日志文件了解系统运行状态和错误信息。

### 2. 检查API响应

使用浏览器开发者工具或Postman等工具，检查API请求和响应，了解API调用情况和错误信息。

### 3. 检查数据库

使用SQLite命令行工具或数据库管理工具，检查数据库中的数据是否正确：

```bash
sqlite3 tbore.db "SELECT * FROM devices;"
sqlite3 tbore.db "SELECT * FROM accounts;"
sqlite3 tbore.db "SELECT * FROM tunnels;"
```

### 4. 检查网络连接

使用ping命令检查服务器和设备之间的网络连接：

```bash
ping localhost
ping 192.168.1.100  # 设备IP
```

### 5. 重启服务

如果系统出现异常，可以尝试重启服务器：

```bash
# 停止服务
pkill -f tbore-server

# 启动服务
./tbore-server
```

## 最佳实践

### 1. 设备管理

- 为每个设备设置唯一的Agent ID
- 定期检查设备状态，及时处理离线设备
- 为重要设备创建多个账户，提高安全性

### 2. 账户管理

- 使用强密码，定期更换密码
- 为不同权限的用户创建不同级别的账户
- 定期清理不再使用的账户

### 3. 隧道管理

- 使用合理的端口映射，避免端口冲突
- 为隧道添加详细的描述，便于管理
- 定期检查隧道状态，及时处理异常隧道

### 4. WebSSH连接

- 使用HTTPS协议访问系统，提高安全性
- 避免在公共网络中使用WebSSH
- 连接完成后及时关闭终端窗口

### 5. 系统维护

- 定期备份数据库
- 定期更新系统版本
- 监控系统资源使用情况
